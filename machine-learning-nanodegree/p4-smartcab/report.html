Answer:

Action Selection Method

There are multiple action selection policies worth mentioning. Boltzmann, ε-greedy and greedy selection. In many simple cases a ε-greedy machanism can be chosen where the ε value can be kept fixed for all number of iterations. Usually, the amount exploration done by the agent should diminish over time, meaning that the policy should get increasingly greedy. This can be achieved making ε approach 0 as the number of trials t increase. A simple form of this could by ε = 1/t. The Boltzmann policy is "softmax" method that uses the Gibbs distribution to estimate the probability of taking an action "a" in a given state "s". In the greedy policy, the agent selects the most interesting actions (as an evaluation function) in a given state. In this case, the agent does not explore all actions.

I think an increasingly greedy policy is the best fit in this case. As the agent needs to learn a fairly large amount of state, action pair values it makes sense to let the agent explore a lot in the beginning before making it more greedy once it has a more complete view of the world. Having a low exploration rate towards the end will also stop the agent from taking any negative reward actions. I have implemented an epsilon-decay action selection function that selects a random action with probability $e^{-γt}$ and otherwise the optimal action according to the policy.

Parameter Tuning

Crucial to successful reinforcement learning is the careful setting of the three meta-parameters α; ε and γ.

For this particular problem we know that the a given (state, action) pair always will give the same results. We can therefore deduce that the optimal learning rate is 1, as this is a fully deterministic environment. [1]

The dicount factor γ determines the importance of the future estimated reward. A factor of 0 will make the agent "short-sighted" by only considering current rewards, while a factor close to 1 will make it strive for a long-term high reward. As we know that the optimal action doesn't depend on the future. The optimal discount factor value is therefore 0.

This leaves us with ε, for which we need to find the optimal value. There are multiple ways to go about tuning the parameters. However, due to the long running time of the simulation (which can be decreased by not displaying graphics) search algorithms such as evolutionary algorithms will take a long time to run, it would also require me to write some intricate fitness function. I have therefore chosen the easy route, namely to select the optimal parameters through trial and error.

Question 5¶

Does your agent get close to finding an optimal policy, i.e. reach the destination in the minimum possible time, and not incur any penalties?

Answer:

My agent seems to learn a policy where it will always take the shortest path dictated by the planner and abide the traffic rules. Since we are using a γ value of 0, the agent will always pick the action that miximizes the reward based on its current state. if the action equals the next waypoint the agent will collect a reward of two, if it moves in any other direction (assuming it follows the traffic rules) it will get a reward of 0.5, and doing no actions will result in a reward of 1. This means that the agent will follow the waypoints and obey the traffic rules, as traffic violations will result in negative rewards (-1). As the agent always picks an action based on the current state alone, and does not factor in future rewards it will always selection action = next waypoint and not not stay put to accumulate additional rewards if it could do so, and still get to the target within the deadline.
